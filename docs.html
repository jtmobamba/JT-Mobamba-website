Quick Start

Get started with JT Mobamba in just 3 steps:

1. Get Your API Credentials

Go to your Dashboard API section, select a collection, and create an API key. You'll receive:

  • API Key - Public identifier (starts with jtm_)
  • Secret Key - Private key (starts with jtms_)
Important: Save your secret key immediately! It will only be shown once.

2. Install the SDK

npm install jtmobamba-mcp
pip install jtmobamba-mcp
flutter pub add jtmobamba_mcp
npm install jtmobamba-mcp-react-native

3. Connect and Query

JavaScript 
const JTMobamba = require('jtmobamba-mcp');

// Initialize the client
const db = new JTMobamba({
    apiKey: 'jtm_your_api_key',
    secretKey: 'jtms_your_secret_key',
    baseUrl: 'https://your-server.com/api/database/v1/data'
});

// Read data
const users = await db.table('users').select();
console.log(users.data);

// Insert data
await db.table('users').insert({
    name: 'John Doe',
    email: 'john@example.com'
});

Authentication

All API requests require authentication using your API Key and Secret Key. Include these headers in every request:

HTTP Headers 
X-API-Key: jtm_your_api_key
X-API-Secret: jtms_your_secret_key
Content-Type: application/json

Permissions

API keys have configurable permissions:

Permission Description Operations
read Read data from tables SELECT, SHOW TABLES, DESCRIBE
insert Insert new rows INSERT
update Update existing rows UPDATE
delete Delete rows DELETE

API Endpoints

Base URL: https://your-server.com/api/database/v1/data

GET /info
Get collection information and list of tables
GET /tables
List all tables in the collection
GET /tables/:tableName
Get table structure and column definitions
GET /tables/:tableName/rows
Read rows from a table (supports pagination and filtering)
POST /tables/:tableName/rows
Insert one or more rows into a table
PUT /tables/:tableName/rows/:rowId
Update a specific row by ID
DELETE /tables/:tableName/rows/:rowId
Delete a specific row by ID
POST /sql
Execute a raw SQL query

CRUD Operations

Create (Insert)

Insert a single row:

JavaScript 
// Insert single row
const result = await db.table('users').insert({
    name: 'John Doe',
    email: 'john@example.com',
    age: 30
});

console.log(result.rowId); // New row ID

Insert multiple rows:

JavaScript 
// Insert multiple rows
const result = await db.table('users').insert([
    { name: 'Alice', email: 'alice@example.com' },
    { name: 'Bob', email: 'bob@example.com' }
]);

console.log(result.rows); // Array of inserted rows

Read (Select)

Query data with filters, pagination, and ordering:

JavaScript 
// Get all rows
const all = await db.table('users').select();

// Filter rows
const filtered = await db.table('users')
    .where({ age: 30 })
    .select();

// Pagination
const paginated = await db.table('users')
    .page(1, 10)  // page 1, 10 items per page
    .orderBy('created_at', 'DESC')
    .select();

// Get first matching row
const user = await db.table('users')
    .where({ email: 'john@example.com' })
    .first();

Update

JavaScript 
// Update by ID
await db.table('users').updateById(1, {
    name: 'Jane Doe',
    age: 31
});

// Update with WHERE conditions
await db.table('users')
    .where({ status: 'pending' })
    .update({ status: 'active' });

Delete

JavaScript 
// Delete by ID
await db.table('users').deleteById(1);

// Delete with WHERE conditions
await db.table('users')
    .where({ status: 'inactive' })
    .delete();

SQL Queries

Execute raw SQL queries for complex operations:

JavaScript 
// SELECT query
const result = await db.sql('SELECT * FROM users WHERE age > 25 ORDER BY name');
console.log(result.result.rows);

// INSERT query
await db.sql(`INSERT INTO users (name, email) VALUES ('John', 'john@example.com')`);

// UPDATE query
await db.sql(`UPDATE users SET status = 'active' WHERE id = 1`);

// DELETE query
await db.sql(`DELETE FROM users WHERE status = 'inactive'`);

Supported SQL Statements

Error Handling

Handle errors gracefully in your application:

JavaScript 
try {
    const users = await db.table('users').select();
} catch (error) {
    if (error.message.includes('Invalid API key')) {
        console.error('Authentication failed');
    } else if (error.message.includes('Rate limit')) {
        console.error('Too many requests, please slow down');
    } else {
        console.error('Error:', error.message);
    }
}

Common Error Codes

Error Cause Solution
Invalid API key API key is incorrect or doesn't exist Check your API key in the dashboard
Invalid secret key Secret key doesn't match the API key Regenerate secret key if lost
Rate limit exceeded Too many requests per hour Wait or increase rate limit
Permission denied API key lacks required permission Update API key permissions
Table not found Table doesn't exist in collection Create the table first
